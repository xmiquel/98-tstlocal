# Change: fastapi-skeleton

## Why

This is the **first real feature slice** in `98-tstlocal`. Three prior cycles (`bootstrap-toolchain`, `ci-quality-gate`, `dev-tooling`) shipped strict Python tooling ‚Äî `uv`, `pytest` with an 80% coverage gate, `ruff` (incl. `S` security rules), `mypy --strict`, an 8-step CI quality gate on `windows-latest`, Dependabot, pre-commit, `Makefile`, and `AGENTS.md` ‚Äî but **zero application code**. The `python-toolchain` capability spec explicitly deferred "FastAPI and the trading application code (next change)" to a follow-up; this proposal is that follow-up's first slice.

The slice delivers the smallest possible FastAPI surface: a tiny app factory in `app/main.py`, a single `GET /health` endpoint returning `{"status": "ok"}`, and a `TestClient` test in `tests/test_health.py`. After it lands, coverage becomes a real signal (no longer vacuous "100% of 0 statements"), the empty `app/` package gets its first executable code, the strict TDD unlock (from `bootstrap-toolchain`) gets exercised on a real feature, and Dependabot starts tracking `fastapi` and `httpx` like the rest of the toolchain. The 8-step CI quality gate is unchanged. ~24 lines of code + a `uv.lock` bump, comfortably under the 400-line review budget.

## What Changes

- **`pyproject.toml`** ‚Äî `dependencies` goes from `[]` to `["fastapi~=0.116", "httpx~=0.27"]` (Option 1 from explore: minimal core install; no `fastapi[standard]`, no `uvicorn` for this slice).
- **`uv.lock`** ‚Äî regenerated by `uv add`, committed in the **same work unit** as the `pyproject.toml` change (precedent: `dev-tooling` apply).
- **`app/main.py`** (NEW) ‚Äî FastAPI app factory + `GET /health` route returning `{"status": "ok"}`; `app: FastAPI = FastAPI()` annotation passes mypy --strict. ~10 lines.
- **`tests/test_health.py`** (NEW) ‚Äî `from fastapi.testclient import TestClient` + assertion on the response body and HTTP 200. ~12 lines.
- **`openspec/specs/python-toolchain/spec.md`** (delta) ‚Äî one **ADDED Requirement** for `Application Runtime`; the existing 8 requirements stay.
- **CI / pre-commit / Dependabot / Makefile / AGENTS.md / README** ‚Äî UNCHANGED. The 8-step `Quality Gate` job is the binding signal; pre-commit's 3 hooks pick up the new files automatically; Dependabot starts tracking `fastapi` and `httpx` on the next weekly run.
- **`app/__init__.py`, `tests/__init__.py`, `tests/test_smoke.py`** ‚Äî UNCHANGED.

## Impact

- **Canonical spec modified**: `python-toolchain` gets a new `Application Runtime` ADDED Requirement. **New full capability spec: none.** The slice is purely additive to the existing capability.
- **First real code, first real coverage signal**: the 100%-of-0-statement vacuity is gone. The 80% coverage gate becomes a meaningful contract for the first time.
- **First strict TDD exercise on a feature**: `apply.tdd: true` (set in `bootstrap-toolchain`) now has a real RED‚ÜíGREEN cycle to validate.
- **Dependency surface grows from 0 ‚Üí 2**: `fastapi` and `httpx` enter the runtime dep graph; ~30 transitive entries (pydantic, starlette, etc.) land in `uv.lock`. Dependabot will start opening weekly PRs for both.
- **What's NOT affected**: no DB, no auth, no frontend, no MT5 integration, no business logic, no `/trade` or `/strategy` endpoints, no models, no persistence, no settings/env loading, no CORS, no middleware, no lifespan.

## Out of Scope (Non-Goals)

Deferred to later slices. Recording these here so future agents reading this proposal don't re-litigate the choices:

- **B. Frontend direction** ‚Äî server-rendered templates, SPA, or none. The trading app's UI is undecided. Not in this slice.
- **C. Persistence choice** ‚Äî SQLite, Postgres, files, none. Trading data lives somewhere eventually; not in this slice.
- **D. Slicing strategy** ‚Äî vertical (one full user journey per slice) vs horizontal (one layer per slice). The next slice after A1 will pick a posture.
- **`fastapi[standard]`, `uvicorn[standard]`, `fastapi-cli`, `jinja2`, `python-multipart`, `email-validator`, `pydantic-settings`, `pydantic-extra-types`** ‚Äî all of FastAPI's "batteries" are deferred. Land them when an actual `make run` target or a real endpoint needs them.
- **A `make run` target** ‚Äî uvicorn isn't in deps; the `Makefile` is unchanged in this slice. Land with the first runnable end-user endpoint.
- **AGENTS.md update for the `httpx 0.27` deprecation warning** ‚Äî cosmetic `DeprecationWarning` on `TestClient(app)`; not a test failure. Document in `AGENTS.md` as a follow-up only if it starts showing up in `pytest -W error` runs.
- **MT5 / Windows-specific runtime deps** ‚Äî land with the MT5 connector slice.
- **Stale `app/__pycache__/cpython-313.pyc`** ‚Äî leftover from a prior 3.13 run; not relevant to this slice. Cleanup is a one-liner.

## Capabilities

### New Capabilities
- **None.** This slice is purely additive to the existing `python-toolchain` capability.

### Modified Capabilities
- `python-toolchain`:
  - **ADDED** `Application Runtime` requirement ‚Äî covers: (a) FastAPI app factory in `app/main.py`; (b) `GET /health` route contract (returns `{"status": "ok"}` with HTTP 200); (c) TestClient-based test pattern; (d) runtime-dep declaration in `pyproject.toml` using `~=` pins (no `uvicorn` yet).

## Approach

Single change, single PR, single squash-merge to `main`. The `pyproject.toml` dep change and the `uv.lock` regeneration land in the **same work unit** (precedent: `dev-tooling` apply). Strict TDD binds: apply phase commits a failing test first (`ModuleNotFoundError: No module named 'fastapi'`), then the implementation that flips it GREEN. No refactor step is expected ‚Äî the slice is small enough that RED‚ÜíGREEN completes the cycle. All tool invocations go through `uv run <tool>`. Conventional commits: `chore(deps)`, `test(health)`, `feat(health)`, `docs(spec)`.

**Decision needed before apply: No.** **Chained PRs recommended: No.** **400-line budget risk: Low** (explore estimate: +24 code lines, +30 lockfile entries).

## Risks

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| `httpx 0.27` emits `DeprecationWarning` on `TestClient(app)` | Med | Cosmetic, not a test failure; document in `AGENTS.md` if it shows up. Or pin `httpx~=0.28` if a clean test output matters more than lockfile churn. |
| `uv.lock` not committed in the same work unit ‚Üí CI `uv lock --check` fails | Low | Work-unit precedent from `dev-tooling`; apply runs `uv lock --check` locally before push. |
| `mypy --strict` chokes on a FastAPI annotation | Low | FastAPI 0.116+ uses `Annotated[...]` (mypy-friendly); fallback is the `pydantic.mypy` plugin. |
| First Dependabot batch opens PRs immediately after merge | Low | Expected; `~=` pins keep bumps patch-only; auto-merge on `Quality Gate` green. |
| Pre-commit hooks (3 existing) catch a real issue on the new files | Low | Hooks exist for this reason; fix and amend. |
| Windows PowerShell `RemoteException` on `uv` commands | Low | Terminal artifact, not a failure; trust the exit code. (Carried over from `AGENTS.md`.) |

## Rollback Plan

Single `git revert` of the squash commit on `main`. Restores `pyproject.toml` to `dependencies = []` and the previous `uv.lock`. Removes `app/main.py` and `tests/test_health.py`. Reverts the `Application Runtime` requirement delta from the canonical spec. The existing 8-step CI quality gate is unaffected ‚Äî rollback proves green. No data, no migrations, no backwards-compat concerns. The `python-toolchain` capability reverts to its post-`dev-tooling` state (still ships the 8 requirements it had before this slice).
