# Proposal: dev-tooling

## Intent

The `ci-quality-gate` verify-report raised **7 SUGGESTION items** plus a **WARNING** that `actions/checkout@v4` and `astral-sh/setup-uv@v3` will be forced to Node 24 on 2026-06-16. This change resolves the deprecation and lands the cross-cutting dev-tooling: Dependabot for actions + Python deps, local pre-commit hooks, a `Makefile` for common commands, dev-dep version tightening (`>=` → `~=`), `AGENTS.md` for future agents/AI, and a CI status badge in the README. Zero application code, zero CI changes.

## Scope

### In Scope
- **4 new files**: `.github/dependabot.yml`, `.pre-commit-config.yaml`, `Makefile`, `AGENTS.md`
- **2 modifications**: `pyproject.toml` (dev-dep version pins), `README.md` (status badge line)
- **1 side-effect**: `uv.lock` regenerated as part of the same work unit as the `pyproject.toml` pin change
- **1 repo-level setup** (USER, 1-time): enable repo auto-merge so Dependabot can self-merge on CI green

### Out of Scope
- **CODEOWNERS** — deferred. Solo-dev repo; would create a self-review loop. SUGGESTION #5 closed with this deferral documented here.
- Any change to `.github/workflows/ci.yml` — the existing 8-step quality gate is the contract
- New application code, FastAPI, MT5, trading domain

## Capabilities

### New Capabilities
- None. All additions live inside the existing `python-toolchain` capability.

### Modified Capabilities
- `python-toolchain`:
  - **MODIFIED** `Toolchain Pinning and Lockfile` — dev-dep constraints `>=` → `~=` (patch updates allowed, breaking bumps blocked)
  - **MODIFIED** `Code Quality (Lint, Format, Type Check)` — add pre-commit hook requirement (local-only)
  - **MODIFIED** `Test Runner and Coverage Gate` — expose `make test` as a thin `uv run pytest` wrapper
  - **ADDED** `Dependabot` — auto-update GitHub Actions and `pyproject.toml` deps; auto-merge on CI green
  - **ADDED** `Pre-commit Hooks` — local-only quality gates (ruff format, ruff check --fix, mypy) before commit
  - **ADDED** `Makefile Targets` — `make help | test | lint | format | format-check | typecheck | lock-check | ci` (all `uv run`-delegated)
  - **ADDED** `AGENTS.md` — project overview, conventions, known gotchas for future AI/agents

## Approach

Single change, single PR, single squash-merge to `main`. The `pyproject.toml` pin change and the `uv.lock` regeneration land in the **same work unit** (per `work-unit-commits`); apply phase re-runs all 8 CI quality commands locally before push. All new files use the same top-of-file comment-block style as `ci.yml` (purpose + spec reference). All tool invocations go through `uv run <tool>`. Conventional commits: `chore(deps)`, `chore(ci)`, `chore(tooling)`, `chore(workspace)`, `docs(readme)`. **Strict TDD** binds: every commit must keep the 8-step CI quality gate green.

## Risks

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Pre-commit hooks bypassable via `--no-verify` | Med | CI mirrors the same checks; bypass is no escape from the contract |
| First Dependabot batch creates PR noise | Med | Auto-merge on CI green + `~=` pins keep bumps patch-only |
| `uv lock` after the pin change fails to resolve | Low | Apply runs `uv lock` locally and re-runs the 8 quality commands before push |
| Repo auto-merge not yet enabled | Low | Documented as a 1-time USER setup step; apply verifies with `gh repo view` |
| `dependencies` label auto-created on first Dependabot PR | Low | Apply pre-creates the label (small, clean) |
| `pre-commit` install hook breaks the dev machine | Low | `pre-commit uninstall` is the documented escape hatch |
| AGENTS.md scope creep | Low | New file; 6-section cap is structural |

## Rollback Plan

Single `git revert` of the change commit. Restores `pyproject.toml` + `uv.lock` to the pre-`~=` state; the 4 new files are simply removed. Any Dependabot PRs already opened are closed with a comment. If the pre-commit install hook breaks the dev machine, `pre-commit uninstall` reverts it. The existing 8-step CI quality gate is unaffected by this change — the contract is preserved.

## Dependencies

- `pre-commit` (PyPI) — added to dev-deps in this change
- `astral-sh/setup-uv@v3`, `actions/checkout@v4` — already in use from `ci-quality-gate`
- Repo setting: `gh repo edit --enable-auto-merge` (USER, 1-time, before merge)

## Alternatives Considered

- **Taskfile** instead of Makefile → rejected: Makefile is more ubiquitous; simplicity wins for a solo-dev project
- **`==` pinning** instead of `~=` → rejected: too rigid for libs; `~=` allows patches while still blocking breaking bumps
- **CI-run pre-commit** instead of local-only → rejected (P2 default): CI already mirrors the checks; pre-commit is for fast feedback, not enforcement
- **Fold AGENTS.md into README.md** → rejected: different audiences (agents/AI vs human users); mixing confuses both
- **Dependabot `pip` ecosystem** instead of `uv` → primary attempt is `uv`; spec phase will verify Dependabot `uv` support and fall back to `pip` if not yet supported
- **Badge in AGENTS.md** instead of README.md → rejected: README is the canonical entry point for humans; AGENTS.md is for AI

## Open Questions

- None blocking. The 4 user-confirmed defaults (A1 Dependabot for actions, A2 Dependabot for pyproject, A3 pre-commit local-only, A4 Makefile) + the `>=` → `~=` pin + the AGENTS.md scaffold + the README badge cover the 7 SUGGESTION items and the Node 20 deprecation WARNING. The 2 non-blockers (auto-merge setup, `dependencies` label) are documented as USER setup steps.

## User Setup Steps (1-time, OUTSIDE this change)

1. Enable repo auto-merge: `gh repo edit --enable-auto-merge` (Dependabot cannot enable auto-merge on its own PRs unless the repo setting allows it)
2. Pre-create the `dependencies` label (optional — Dependabot will create it on its first PR if missing): `gh label create dependencies --color "0366d6" --description "Dependabot dependency updates"`
