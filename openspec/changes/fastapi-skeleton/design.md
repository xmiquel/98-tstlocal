# Design: fastapi-skeleton

## Context

The first real feature slice in `98-tstlocal`. Three prior cycles
(`bootstrap-toolchain`, `ci-quality-gate`, `dev-tooling`) shipped
strict Python tooling but **zero application code**. This slice
lands the smallest possible FastAPI surface — a module-level app in
`app/main.py`, one `GET /health` route, and a `TestClient` contract
test — so the 80% coverage gate stops being vacuous (100% of 0
statements) and the strict TDD unlock (from `bootstrap-toolchain`)
gets exercised on a real feature for the first time.

Scope is locked by `proposal.md` and `specs/python-toolchain/spec.md`:

- `pyproject.toml` `dependencies` → `["fastapi~=0.116", "httpx~=0.27"]`
- `app/main.py` (NEW) — module-level `app: FastAPI = FastAPI()` + one route
- `tests/test_health.py` (NEW) — `TestClient` contract test
- `uv.lock` — regenerated, committed in the same work unit as `pyproject.toml`
- Canonical spec gets one ADDED Requirement (`Application Runtime`)
- ~24 lines of code + a lockfile bump; comfortably under the 400-line budget

## Module: `app/main.py` (NEW)

```python
"""Minimal FastAPI application for the 98-tstlocal trading app skeleton.

The full app (strategies, backtest, MT5 connector) lands in subsequent
changes. This module exists today so the runtime-dep toolchain can be
exercised end-to-end on a real feature: a single GET /health route that
returns a fixed status payload, exercised by tests/test_health.py.
"""

from fastapi import FastAPI

app: FastAPI = FastAPI()


@app.get("/health")
def health() -> dict[str, str]:
    """Health-check endpoint; returns a fixed status payload."""
    return {"status": "ok"}
```

Notes:

- Module-level `app` (not a `create_app()` factory) — matches the canonical
  FastAPI skeleton pattern, the proposal's explicit choice, and the spec's
  `Application Runtime` requirement. A factory is the right move the moment
  the project grows config-driven startup (settings, lifespan, multiple
  workers); for a 10-line skeleton, the indirection is pure overhead.
- `app: FastAPI = FastAPI()` is annotated so the declaration passes
  `mypy --strict` without a plugin. FastAPI 0.116+ uses `Annotated[...]`
  internally for route handlers, so the call site annotations are not
  exposed to user code in this slice.
- `def health() -> dict[str, str]` — typed return satisfies `mypy --strict`;
  `B008` (FastAPI function call in default arg) does not apply because
  the route handler is called by FastAPI, not used as a default.

## Module: `tests/test_health.py` (NEW)

```python
"""Contract test for the GET /health endpoint."""

from fastapi.testclient import TestClient

from app.main import app

client = TestClient(app)


def test_health_returns_ok_status() -> None:
    response = client.get("/health")
    assert response.status_code == 200
    assert response.json() == {"status": "ok"}
```

Notes:

- `TestClient(app)` is a synchronous test client backed by httpx; it does
  not require uvicorn or any running event loop. The slice does not add
  `uvicorn` as a runtime dep — uvicorn is the server, TestClient is the
  test driver, and they are different concerns.
- `assert` is allowed here because `pyproject.toml` has
  `"tests/**" = ["S101"]` in `[tool.ruff.lint.per-file-ignores]`.
- The test function is typed `-> None` for `mypy --strict`.
- `client` is module-level on purpose: `TestClient.__init__` walks the
  FastAPI route table once; reusing the instance across tests is the
  recommended pattern in FastAPI's own docs.

## `pyproject.toml` Delta

```diff
 [project]
 name = "98-tstlocal"
 version = "0.1.0"
 description = "FastAPI trading app — strict Python toolchain bootstrapped before application code."
 requires-python = ">=3.12,<3.13"
-dependencies = []
+dependencies = [
+    "fastapi~=0.116",
+    "httpx~=0.27",
+]
```

Both pins are `~=` (compatible release). Per PEP 440, a 2-component
`~=X.Y` resolves to `>=X.Y, <(X+1).0` — so `~=0.116` is `>=0.116, <1.0`
and `~=0.27` is `>=0.27, <1.0`. For pre-1.0 packages, that makes the
`~=` pin a "no accidental 1.0" guard rather than a patch-only clamp.
This matches the existing dev-dep convention (`pytest~=9.0`, `mypy~=2.1.0`,
etc.) where the pin is a major-line fence. Dependabot's weekly batches
propose the latest version within the pinned range; auto-merge on
`Quality Gate` green handles them.

Both go in `[project] dependencies` (runtime), **not** in
`[dependency-groups].dev` (dev-only). `fastapi` is the runtime framework
the application imports; `httpx` is required at test-time by `TestClient`
and is therefore a runtime transitive of the test suite, but the
canonical placement is `dependencies` so the dep graph is honest about
what the package needs to import.

## `uv.lock` Handling

`uv add fastapi~=0.116 httpx~=0.27` regenerates `uv.lock` with the new
transitive graph (~30 entries: pydantic, starlette, anyio, h11, idna,
sniffio, typing-extensions, …). `uv.lock` MUST be committed in the
**same work unit** as the `pyproject.toml` change — the binding CI step
`uv lock --check` will fail otherwise. Precedent: `dev-tooling` apply
landed dep pins and the lockfile together; this slice follows the same
precedent.

## Commit Plan (Strict TDD)

Four commits, one PR, squash-merge to `main`. RED→GREEN→document.

| # | Conventional commit | Files | Why this order |
|---|----------------------|-------|----------------|
| 1 | `chore(deps): add fastapi~=0.116 and httpx~=0.27` | `pyproject.toml`, `uv.lock` | Deps land first so subsequent commits can import `fastapi` and `httpx` without a broken `uv` environment. Lockfile in same work unit (CI binding step). |
| 2 | `test(health): add RED test for GET /health` | `tests/test_health.py` | The new test imports `from app.main import app`, which does not exist yet. `uv run pytest` fails with `ModuleNotFoundError: No module named 'app.main'`. RED committed, no skipped tests, no `xfail`. |
| 3 | `feat(health): add minimal FastAPI app with /health route` | `app/main.py` | Adds the module that commit 2 imports. `uv run pytest` flips GREEN. The /health contract is now exercised by the test suite. |
| 4 | `chore(spec): include openspec delta for fastapi-skeleton` | `openspec/changes/fastapi-skeleton/{proposal.md, specs/, design.md, tasks.md}` | The spec arc closes on the same PR as the code that satisfies it. The archive phase will read the delta from this commit and merge it into the canonical spec. |

Strict TDD evidence (verifiable from `git log --oneline`): the
`test(health):` commit appears before the `feat(health):` commit. No
squash-merge hides this — the PR is squash-merged, but the four
intermediate commits remain on the feature branch and are visible via
`git log --oneline` on the branch ref before the PR is closed.

Refactor step: not expected. The slice is small enough that RED→GREEN
completes the cycle; a follow-up slice (e.g., when a real endpoint needs
config-driven startup) is the natural place to extract a `create_app()`
factory.

## CI Implications

All 8 steps of the `Quality Gate` job on `windows-latest` fire:

| # | Step | What runs | Why it passes |
|---|------|-----------|----------------|
| 1 | `actions/checkout@v6` | checkout | unchanged |
| 2 | `astral-sh/setup-uv@v7` | install uv | unchanged |
| 3 | `actions/setup-python@v5` | pin Python 3.12 | unchanged |
| 4 | `uv sync` | install deps including fastapi+httpx | `pyproject.toml` + `uv.lock` in commit 1 |
| 5 | `uv run ruff check .` | lint | `app/main.py` and `tests/test_health.py` use bare `assert` only inside `tests/**` (S101 ignored) and follow PEP 8 / isort (`tests/**` and `app/` are both `known-first-party`) |
| 6 | `uv run ruff format --check .` | format | both new files written in the canonical double-quote / space-indent style from day one |
| 7 | `uv run mypy .` | strict type check | `app: FastAPI` annotation; `def health() -> dict[str, str]`; `def test_health_returns_ok_status() -> None`; no implicit Optional |
| 8 | `uv run pytest` | run tests with `--cov=app --cov-fail-under=80` | commit 3 makes the test GREEN; coverage on `app/` is 100% (1 file, ~5 statements, 1 hit) |

The `dependencies` and `dependencies-dev` jobs in CI are unaffected
(this slice touches `dependencies`, not `dependency-groups`).

## Risks & Mitigations

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| `httpx 0.27` emits a `DeprecationWarning` on `TestClient(app)` construction | Med | Cosmetic, not a test failure; documented in `proposal.md` and `spec.md` non-goals. AGENTS.md follow-up only if it starts showing up under `pytest -W error`. Pin `~=0.28` only if the warning becomes noisy. |
| `uv.lock` not committed in the same work unit as `pyproject.toml` → `uv lock --check` fails | Low | Work-unit precedent from `dev-tooling`; apply runs `uv lock --check` locally before push. Commit 1 of the plan above bundles them. |
| `mypy --strict` chokes on a FastAPI annotation | Low | FastAPI 0.116+ uses `Annotated[...]` internally; user-visible code in this slice uses bare `dict[str, str]` and `None`, which mypy handles. Fallback: add `pydantic.mypy` plugin to `[tool.mypy]`. |
| First Dependabot batch opens PRs for `fastapi` and `httpx` within a week of merge | Low (expected) | `~=` is a major-line fence (per PEP 440: `~=0.116` = `>=0.116, <1.0`); Dependabot proposes the latest within the range, not patch-only. Auto-merge on `Quality Gate` green handles them; no manual triage needed. |
| Pre-commit hooks catch a real issue on the new files | Low | Hooks exist for this reason (ruff-pre-commit v0.6.9, ruff-format, local mypy). Fix and amend. |
| `tests/test_health.py` shadows the `app` symbol locally | Low | The `app` in `TestClient(app)` is the imported FastAPI instance; the function param shadowing only happens inside the test function and does not collide. |

## Out of Scope (Non-Goals)

Recording here so the apply phase and any future agent reading this
design do not re-litigate them:

- `uvicorn` runtime dep and any `make run` target
- Application lifespan / startup / shutdown events
- Middleware (CORS, GZip, trusted hosts, request ID)
- Persistence (database, ORM, file storage, settings)
- Auth (API keys, OAuth2, JWT, sessions)
- Additional endpoints beyond `GET /health`
- Frontend, templates, static files, `fastapi[standard]` extras
- The `httpx 0.27` `DeprecationWarning` cosmetic fix
- MT5 / Windows-specific runtime deps
- Refactor of the module-level `app` into a `create_app()` factory
- Updating `AGENTS.md` to mention the new `app/main.py` and the
  `TestClient` pattern (deferred to the slice that adds a second
  endpoint, when the pattern is no longer "one example")
